# 数据收集指南

## 概述

本指南说明如何收集论文、代码或资源的相关信息。

## 选题原则

选择内容时需要考虑：
- **研究价值**：是否是前沿、有影响力的研究方向
- **实用性**：是否对实际开发或研究有帮助
- **完整性**：是否有足够的资料可以整理
- **时效性**：是否是近期（1-2年内）的工作

## 数据收集清单

### 论文类型内容

需要收集以下信息（如果用户未提供，保持空缺）：

- [ ] arXiv 论文编号（如 2501.09136）
- [ ] 论文标题（中英文）
- [ ] 论文作者信息
- [ ] 论文摘要（中英文）
- [ ] 论文发布日期
- [ ] arXiv 主题分类（如 cs.AI, cs.CL, cs.IR）
- [ ] 论文 PDF 链接
- [ ] 论文官方代码仓库（如果有）
- [ ] 相关博客、笔记、解读文章
- [ ] 论文的核心贡献和创新点
- [ ] 实验结果和性能数据
- [ ] 方法的技术细节

### 代码类型内容

需要收集以下信息（如果用户未提供，保持空缺）：

- [ ] GitHub 仓库名称（如 asinghcsu/AgenticRAG-Survey）
- [ ] 仓库描述（中英文）
- [ ] 星标数量
- [ ] 主要标签（如 reflection, tools, multiagent）
- [ ] README 文档内容
- [ ] 主要功能特性
- [ ] 使用示例
- [ ] 相关论文（如果有）
- [ ] 相关资源链接

### 资源类型内容

需要收集以下信息（如果用户未提供，保持空缺）：

- [ ] 资源标题（中英文）
- [ ] 资源链接
- [ ] 资源描述（中英文）
- [ ] 资源类型（数据集、评测基准、教程等）
- [ ] 使用说明
- [ ] 相关论文或代码
- [ ] 适用场景

## 数据收集方法

### arXiv 论文信息收集

使用 arXiv API 获取论文信息：

```bash
curl "https://export.arxiv.org/api/query?id_list=2501.09136"
```

### GitHub 仓库信息收集

使用 GitHub API 获取仓库信息：

```bash
curl "https://api.github.com/repos/asinghcsu/AgenticRAG-Survey"
```

### 相关资源搜索

- Google Scholar 搜索引用论文
- GitHub 搜索相关实现
- Medium、知乎等平台搜索解读文章
- arXiv 搜索相关论文

## 注意事项

1. **数据真实性**：如果用户未提供数据，不要编撰，保持空缺
2. **信息准确性**：确保收集的信息准确无误
3. **来源可靠性**：优先使用官方来源（arXiv、GitHub 官方页面）
4. **时效性**：检查信息的时效性，确保是最新的
5. **完整性**：尽可能收集完整的信息，但不要编造
