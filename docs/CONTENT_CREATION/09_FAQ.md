# 常见问题

## 概述

本指南解答内容创建过程中的常见问题。

## 内容类型相关

### Q1: 如何确定内容的类型？

**A**: 根据内容的主要属性：
- 有 arXiv 论文编号 → 论文类型
- 有 GitHub 仓库 → 代码类型
- 有相关资源链接 → 资源类型
- 可以同时属于多个类型

### Q2: 一个内容可以同时属于多个类型吗？

**A**: 可以。一个内容可以同时包含 arxiv、github 和 related 字段，从而同时出现在 papers、code 和 resources 页面。

### Q3: 如果内容没有明确的类型，该如何分类？

**A**: 根据内容的主要属性选择最合适的类型。如果仍然不确定，可以选择资源类型。

## 数据收集相关

### Q4: 如果用户未提供某些数据，应该如何处理？

**A**: 如果用户未提供数据，不要编撰，保持空缺。只使用用户提供的数据。

### Q5: 如何确保收集的数据准确？

**A**:
- 优先使用官方来源（arXiv、GitHub 官方页面）
- 使用 API 获取数据
- 交叉验证数据来源
- 检查数据的时效性

### Q6: 如何收集相关资源？

**A**:
- 使用 Google Scholar 搜索引用论文
- 使用 GitHub 搜索相关实现
- 在 Medium、知乎等平台搜索解读文章
- 在 arXiv 搜索相关论文

## 翻译相关

### Q7: 翻译时如何处理专业术语？

**A**: 参考术语翻译对照表，保持一致性。如果遇到新术语，先确定合适的中文翻译，然后更新术语对照表。

### Q8: 如何确保翻译质量？

**A**:
- 进行多轮校对
- 重点检查技术术语的准确性
- 确保中英文描述在语义上一致
- 避免直译，注重意译

### Q9: 翻译时是否需要保持原文格式？

**A**: 是的，需要保持原文格式，包括段落结构、列表、标题等。

## 内容撰写相关

### Q10: 内容长度超过 100 行怎么办？

**A**: 100 行限制仅适用于代码文件，MDX 文档不限制长度。但建议保持简洁，避免内容过长。

### Q11: 如何确保链接的有效性？

**A**:
- 定期检查链接
- 优先使用稳定的链接源（如 arXiv、GitHub 官方页面）
- 使用相对路径引用站内内容
- 提供备用链接

### Q12: 内容撰写时应该注意什么？

**A**:
- 使用清晰的标题和结构
- 保持语言简洁、专业
- 遵循 Markdown 格式规范
- 确保所有链接有效

## 标签管理相关

### Q13: 如何选择合适的标签？

**A**:
- 参考分类标签清单
- 选择与内容最相关的标签
- 使用已有标签，避免创建新标签
- 标签数量控制在 2-5 个

### Q14: 如果需要添加新标签，应该如何处理？

**A**:
1. 检查是否已有类似标签
2. 评估标签的通用性和复用性
3. 在分类标签清单中添加新标签
4. 更新分类标签管理文档

### Q15: 标签格式有什么要求？

**A**:
- 使用小写字母和连字符（kebab-case）
- 示例：`agentic-rag`, `llm-agent`, `tool-use`

## 内容审核相关

### Q16: 审核时应该重点关注什么？

**A**:
- 技术准确性
- 术语一致性
- 链接有效性
- 格式规范性
- 内容完整性

### Q17: 审核流程是怎样的？

**A**:
1. 自审：作者自行检查
2. 互审：至少一人进行交叉审核
3. 终审：负责人进行最终审核

### Q18: 审核发现问题时应该如何处理？

**A**:
- 记录发现的问题
- 分类整理问题
- 及时修改问题
- 修改后再次审核

## 文件创建相关

### Q19: 文件命名有什么要求？

**A**:
- 使用小写字母和连字符（kebab-case）
- 文件名要简洁、有代表性
- 示例：`agentic-rag-survey.mdx`

### Q20: 文件应该放在哪里？

**A**: 文件必须放在 `src/content` 目录下。

### Q21: 提交信息格式有什么要求？

**A**: 使用规范的提交信息格式：

```
docs: 添加 [内容标题]

- 添加 [内容类型] 内容
- 包含 [主要特性]
- 关联 [相关资源]
```

## 内容更新相关

### Q22: 如何处理内容更新？

**A**: 当有新版本或新发现时：
- 更新元数据（如星标数、新版本）
- 添加新的相关资源
- 更新内容描述

### Q23: 如何确保内容的时效性？

**A**:
- 定期检查链接有效性
- 及时更新过时信息
- 添加新的相关资源
- 关注最新研究进展

## 工具使用相关

### Q24: 有哪些数据收集工具可以使用？

**A**:
- arXiv API: https://export.arxiv.org/api/query
- GitHub API: https://api.github.com
- Google Scholar: https://scholar.google.com

### Q25: 有哪些翻译工具可以使用？

**A**:
- DeepL: https://www.deepl.com
- Google Translate: https://translate.google.com

### Q26: 有哪些格式检查工具可以使用？

**A**:
- Markdown Lint: https://github.com/markdownlint/markdownlint

## 其他问题

### Q27: 如何获取更多帮助？

**A**:
- 查阅相关文档
- 查看示例内容
- 联系团队成员
- 搜索相关资源

### Q28: 如何报告问题或提出建议？

**A**:

**A**:
- 定期检查文档
- 根据实际情况更新文档

## 注意事项

1. **数据真实性**：如果用户未提供数据，不要编撰，保持空缺
2. **术语一致性**：使用统一的术语翻译对照表
3. **内容简洁性**：代码文件不超过 100 行，MDX 文档不限制
4. **标签规范性**：使用已有标签，避免创建新标签
5. **质量可控性**：按照审核清单逐项检查
6. **链接有效性**：确保所有链接有效
7. **时效性**：定期更新内容，保持时效性
