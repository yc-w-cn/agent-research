# 内容审核指南

## 概述

本指南说明如何按照审核清单检查内容质量。

## 审核清单

### 元数据审核

#### 基本元数据

- [ ] title 是否准确、简洁
- [ ] categories 是否正确、完整
- [ ] date 格式是否正确（YYYY-MM-DD）

#### arXiv 元数据（如果有）

- [ ] arxiv.id 是否正确
- [ ] arxiv.subjects 是否完整
- [ ] arxiv.description 中英文是否一致

#### GitHub 元数据（如果有）

- [ ] github.name 是否正确
- [ ] github.description 是否清晰
- [ ] github.stars 是否准确
- [ ] github.tags 是否合理

#### 相关资源（如果有）

- [ ] related 资源链接是否有效
- [ ] related 资源标题是否准确
- [ ] related 资源描述是否清晰

### 内容审核

#### 概述部分

- [ ] 概述是否简明扼要
- [ ] 背景介绍是否清晰
- [ ] 问题说明是否准确

#### 核心内容部分

- [ ] 核心内容是否逻辑清晰
- [ ] 结构是否合理
- [ ] 表达是否准确

#### 主要贡献部分

- [ ] 主要贡献是否突出
- [ ] 贡献点是否准确
- [ ] 是否按重要性排序

#### 关键特性部分

- [ ] 关键特性是否准确
- [ ] 技术特点是否清晰
- [ ] 创新之处是否明确

#### 相关资源部分

- [ ] 相关资源是否完整
- [ ] 链接是否有效
- [ ] 描述是否清晰

### 技术审核

- [ ] 技术描述是否准确
- [ ] 术语翻译是否统一
- [ ] 链接是否有效
- [ ] 代码示例是否正确

### 格式审核

- [ ] 内容长度是否合理（代码文件不超过 100 行）
- [ ] 语言表达是否流畅、专业
- [ ] 格式是否符合规范
- [ ] 标题层级是否正确

## 审核流程

### 1. 自审

作者自行检查，按照审核清单逐项检查。

### 2. 互审

至少一人进行交叉审核，重点检查：
- 技术准确性
- 语言表达
- 格式规范

### 3. 终审

负责人进行最终审核，确认：
- 内容质量
- 格式规范
- 链接有效

## 审核要点

### 数据真实性

- 如果用户未提供数据，不要编撰，保持空缺
- 确保所有数据准确无误

### 术语一致性

- 使用统一的术语翻译对照表
- 保持术语翻译一致

### 内容详实性

- MDX 文档不限制长度，需要非常详实，包含所有核心内容

### 标签规范性

- 使用已有标签，避免创建新标签
- 标签数量合理（2-5 个）

### 质量可控性

- 按照审核清单逐项检查
- 确保所有审核项都通过

## 常见问题

### 问题 1：元数据不完整

**解决方案**：检查元数据字段，确保所有必填字段都已填写。

### 问题 2：链接失效

**解决方案**：检查所有链接，确保链接有效。

### 问题 3：术语不统一

**解决方案**：使用术语翻译对照表，保持术语翻译一致。

### 问题 4：内容过短

**解决方案**：增加内容，确保覆盖所有核心问题。

### 问题 5：格式不规范

**解决方案**：遵循 Markdown 格式规范，确保格式正确。

## 注意事项

1. **逐项检查**：按照审核清单逐项检查，不要遗漏
2. **重点检查**：重点检查技术准确性和链接有效性
3. **多次审核**：进行多轮审核，确保质量
4. **记录问题**：记录发现的问题，及时修改
5. **确认修改**：修改后再次审核，确保问题已解决
