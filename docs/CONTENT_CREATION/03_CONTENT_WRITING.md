# 内容撰写指南

## 概述

本指南说明如何按照规范撰写内容，确保结构清晰、内容准确。

## 内容结构设计

每个内容文件应包含以下部分：

```mdx
---
# 前置元数据
title: 标题
categories:
  - 分类1
  - 分类2
date: YYYY-MM-DD
arxiv: # 可选
  id: 论文编号
  subjects:
    - cs.AI
  description:
    zh: 中文描述
    en: 英文描述
github: # 可选
  name: 仓库名称
  description: 仓库描述
  stars: 星标数
  tags:
    - 标签1
related: # 可选
  - title: 相关资源标题
    url: 链接
    description: 描述
---

## 概述
简要介绍内容的背景和意义

## 核心内容
详细说明主要内容

### 主要贡献
列出主要贡献点

## 关键特性
说明关键特性

## 相关资源
列出相关资源链接
```

## 内容撰写要点

### 概述部分

- 简明扼要地介绍背景
- 说明问题的重要性
- 引出主要内容

### 核心内容部分

- 逻辑清晰，层次分明
- 使用小标题组织内容
- 适当使用列表说明要点

### 主要贡献部分

- 列出 3-5 个主要贡献
- 每个贡献用一句话说明
- 按重要性排序

### 关键特性部分

- 说明技术特点
- 解释创新之处
- 对比传统方法

### 相关资源部分

- 提供论文、代码、博客等链接
- 简要说明每个资源的价值

## 内容质量控制

- **长度控制**：代码文件不超过 100 行，MDX 文档不限制
- **语言规范**：使用简洁、专业的中文
- **格式规范**：遵循 Markdown 格式规范
- **链接检查**：确保所有链接有效

## 注意事项

1. **数据真实性**：如果用户未提供数据，不要编撰，保持空缺
2. **结构清晰**：使用清晰的标题和结构
3. **语言简洁**：避免冗余，保持简洁
4. **格式规范**：遵循 Markdown 格式规范
5. **链接有效**：确保所有链接有效
