# 内容核心问题维度

## 概述

本指南说明不同类型的内容需要涵盖哪些核心问题。

## 论文类型内容

### 1. 研究背景与动机

- 为什么这个问题重要？
- 现有方法的局限性是什么？
- 本文要解决什么问题？

### 2. 核心贡献与创新点

- 提出了什么新方法？
- 有哪些技术创新？
- 相比现有方法的优势是什么？

### 3. 方法细节

- 方法的基本原理
- 关键技术组件
- 系统架构设计

### 4. 实验结果

- 实验设置和数据集
- 主要实验结果
- 与现有方法的对比
- 消融实验分析

### 5. 应用场景

- 方法适用于哪些场景？
- 实际应用案例
- 局限性和适用范围

### 6. 未来方向

- 可以进一步改进的地方
- 潜在的研究方向

## 代码类型内容

### 1. 项目简介

- 项目的主要功能
- 解决什么问题
- 适用场景

### 2. 核心特性

- 主要功能特性
- 技术亮点
- 与其他项目的对比

### 3. 技术架构

- 系统架构设计
- 主要模块说明
- 技术栈

### 4. 使用指南

- 安装方法
- 基本使用示例
- 配置说明

### 5. 实践案例

- 典型应用场景
- 使用示例
- 最佳实践

### 6. 扩展与定制

- 如何扩展功能
- 定制化方法
- 集成建议

## 资源类型内容

### 1. 资源概述

- 资源类型和用途
- 资源规模和特点
- 适用领域

### 2. 资源详情

- 数据集：数据规模、格式、标注方式
- 评测基准：评测任务、指标、基线
- 教程：内容大纲、难度级别、前置知识

### 3. 使用指南

- 如何获取资源
- 使用方法和步骤
- 注意事项

### 4. 相关工作

- 相关论文
- 相关代码
- 其他资源

### 5. 应用案例

- 典型应用
- 成功案例
- 实践经验

## 注意事项

1. **数据真实性**：如果用户未提供数据，不要编撰，保持空缺
2. **内容完整性**：确保涵盖所有核心问题
3. **逻辑清晰**：确保内容逻辑清晰，层次分明
4. **重点突出**：突出核心问题和关键信息
5. **实用性强**：确保内容具有实际参考价值

## 示例

### 论文类型示例

```mdx
## 概述

Agentic-RAG explores advanced Retrieval-Augmented Generation systems enhanced with AI LLM agents.

## 核心内容

本文系统性地回顾了 Agentic RAG 领域的研究进展，探讨了如何通过 Agent 增强传统的 RAG 系统。

### 主要贡献

1. 系统性梳理了 Agentic RAG 的研究脉络
2. 提出了 Agentic RAG 的分类框架
3. 分析了不同方法的优缺点
4. 展望了未来的研究方向

## 关键特性

### 反思机制
Agent 能够对检索和生成过程进行反思，持续优化结果。

### 工具使用
通过调用外部工具增强检索和生成能力。

### 多 Agent 协作
多个 Agent 协同工作，分工处理不同任务。
```

### 代码类型示例

```mdx
## 概述

AgenticRAG-Survey 是一个探索智能体增强检索增强生成系统的项目。

## 核心内容

该项目提供了 Agentic RAG 系统的完整实现，包括反思、规划、工具使用和多代理协作等功能。

### 主要特性

1. 支持多种 Agentic 设计模式
2. 提供完整的实验框架
3. 包含丰富的示例代码
4. 支持多种 LLM 后端

## 关键特性

### 反思机制
实现了 Agent 的自我反思能力，能够持续优化检索和生成结果。

### 工具使用
支持调用外部工具，扩展 Agent 的能力。

### 多 Agent 协作
支持多个 Agent 协同工作，提高系统效率。
```

### 资源类型示例

```mdx
## 概述

AgentBench 是一个评估大语言模型智能体能力的评测基准。

## 核心内容

AgentBench 提供了全面的评测框架，用于评估 LLM Agent 在各种任务中的表现。

### 主要特点

1. 涵盖多个评测维度
2. 提供标准化的评测流程
3. 包含丰富的评测任务
4. 支持多种 LLM 模型

## 关键特性

### 评测维度
包括推理能力、工具使用能力、规划能力等多个维度。

### 评测任务
提供多样化的评测任务，覆盖不同应用场景。

### 评测指标
提供标准化的评测指标，便于对比分析。
```
